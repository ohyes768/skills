# Update Doc - 项目文档自动更新技能

## 功能说明

根据项目的代码改动，自动识别并更新所有相关文档，确保文档与代码实现保持同步。

## 支持的文档类型

- **产品需求文档 (PRD)** - `docs/产品需求文档(PRD).md`
- **技术设计文档 (TDD)** - `docs/技术设计文档(TDD).md`
- **API接口文档** - `docs/API接口文档.md`
- **数据字典** - `docs/数据字典.md`
- **文档索引** - `docs/文档索引.md`

## 使用方法

### 在 Claude Code 中调用

**方式 1：自然语言描述**
```
更新项目文档
根据代码改动更新文档
同步文档和代码
```

**方式 2：直接调用技能**
```
update-doc
```

## 工作流程

1. **分析改动** - 查看最近的 git 历史和未提交的改动
2. **识别影响** - 判断改动对文档的影响范围
3. **更新文档** - 根据改动类型更新相应的文档章节
4. **验证质量** - 检查文档的准确性和完整性
5. **提交更新** - 将文档更新提交到 git

## 典型使用场景

### 场景 1：新增功能
完成新功能开发后，运行技能更新 PRD、TDD、API 文档等。

### 场景 2：API 变更
修改或新增 API 后，自动更新 API 接口文档。

### 场景 3：数据模型变更
数据库表结构调整后，更新数据字典。

### 场景 4：代码重构
架构调整后，更新技术设计文档。

## 注意事项

- ✅ 确保项目有标准的文档结构（`docs/` 目录）
- ✅ 代码改动后及时更新文档
- ✅ 保持文档风格统一
- ⚠️ 大规模重构时建议分批更新文档

## 文档更新原则

1. **及时性** - 代码改动后立即更新
2. **准确性** - 文档必须真实反映代码
3. **完整性** - 更新所有受影响的文档
4. **一致性** - 保持文档风格统一
5. **可追溯性** - 记录变更日期和内容
